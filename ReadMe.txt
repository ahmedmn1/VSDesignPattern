Design Pattern Categories:-
- Behavioral:- Offer ways to manage state of the objects and let objects better communicate with each other. 
- Structural:- Offer ways to compose the objects using inheritance and otherwise, to facilitate a new and useful functionality.
- Creational:- Offer ways to create objects without newing them up yourself, this offer flexibility by letting the conusumer determine
  the approperiate objects to create

Design Pattern:- 
 Are object oriented techniques best practices that have been proven to work in the past. They are not a silver bullet, but they can help you write better code.
